v2 Build Plan — Workouts, Noise, Mindfulness

2026-07-10 · v1 deployed and validated on real data (steps dedupe fixed after real-device check)

v2 adds the three deferred features: workout detail (route map + in-workout heart rate), noise exposure, and mindful minutes. Same TDD discipline; data model grows one column.

Decisions
(1) Maps: Leaflet + OpenStreetMap tiles — only new frontend dependency; hand-rolled maps fail the sanity bar, SVG-only routes lose all context. Tiles need internet; fine for a LAN dashboard. (2) Routes stored as downsampled [lat, lon] lists (≤500 points) in a new workouts.route_json column — added via idempotent ALTER TABLE migration, no re-import required for old rows (they simply lack routes until next import). (3) In-workout HR curve queried from raw samples between workout start/end — no new storage. (4) Workouts UI = fifth domain row (recent list → expand one → map + HR curve), Noise & mindfulness = sixth row with daily charts. No new pages.
