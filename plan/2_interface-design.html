Interface Design

2026-07-07 · interactive mockup: design/mockup.html (open in browser) · screenshots: design/screenshot-light.png, design/screenshot-dark.png

"Daily brief" layout chosen over dense-grid and timeline-journal alternatives: a 30-second morning glance up top, archive depth one click away. Verified as a working interactive mockup with sample data in light and dark modes.

Decision
Layout = daily brief: score strip (readiness hero + sleep score + weekly exercise) → global time-range filter (7d/30d/90d) → four expandable domain rows (Sleep, Heart, Activity, Body & vitals), each with headline numbers + 30-day sparkline collapsed, full charts expanded.

Visualization rules applied (dataviz method)

  • Palette: reference dataviz palette; sleep-stage ordinal ramps validated by script for light (#86b6ef→#184f95) and dark (#9ec5f4→#184f95) surfaces — all checks pass.
  • Marks: bars ≤24px with 4px rounded data-ends + 2px surface gaps; 2px lines; end-dot with surface ring; hairline solid gridlines; area washes only.
  • Every chart: hover tooltip + crosshair, table-view twin (collapsible), selective end-labels only; text never wears series color.
  • Dark mode: own palette steps, not a flip; toggle + prefers-color-scheme default.
  • Status green reserved for sync-ok dot and positive deltas; never used as a series.
Carried into build (v1 frontend spec)
The mockup is the frontend spec: React + Recharts implementation should reproduce this layout, palette, and interaction contract against real API data. Mockup's inline-SVG charts map to Recharts equivalents (or stay hand-rolled SVG if Recharts fights the band/gap specs — decide in build).
